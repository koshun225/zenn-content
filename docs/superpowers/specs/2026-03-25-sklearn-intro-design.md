# sklearn入門記事 設計書

## 概要

Pythonエンジニア向けのscikit-learn入門記事。Wineデータセットを使った分類タスクを題材に、段階的にステップアップしながら機械学習の基本フローを体験するチュートリアル。

## ターゲット読者

- Pythonは書けるが機械学習は初めて
- 「sklearnを使って何かやってみたい」というモチベーション

## アプローチ

**段階的ステップアップ型**: 最初は素朴な「fit → predict」から始めて、段階的に前処理・評価・チューニングを追加。各ステップで精度の変化を見せ、「なぜその工程が必要か」を体感させる。

## 記事構成

### 1. はじめに（約300字）

- scikit-learnとは何か（1〜2文で簡潔に）
- この記事で何ができるようになるか
- 使用するデータセット（Wine）の紹介

### 2. Step1: まず動かす（約800字）

- `load_wine()` でデータ読み込み
- `train_test_split` で訓練データ・テストデータに分割
- `RandomForestClassifier` をデフォルト設定で `fit` → `predict`
- `accuracy_score` で精度確認（0.93〜0.96程度を想定）
- **ゴール**: 最小限のコードで「動いた！」という成功体験

### 3. Step2: 前処理を入れる（約800字）

- 問題提起: Wineデータの特徴量はスケールがバラバラ
- `StandardScaler` で標準化してから同じモデルで再学習
- 精度の変化を確認
- ランダムフォレストはスケーリングに頑健なため劇的な変化は起きにくい点を正直に記述
- 補足: SVMやロジスティック回帰ではスケーリングの効果が大きいと一言添える
- **ゴール**: 前処理の意味と必要性を理解する

### 4. Step3: モデルを評価する（約800字）

- accuracyだけでは不十分という話（クラス不均衡の例を軽く触れる）
- `confusion_matrix` で混同行列を出力
- `classification_report` でprecision, recall, f1-scoreを確認
- 「どのクラスを間違えやすいか」を読み解く
- **ゴール**: 適切なモデル評価ができるようになる

### 5. Step4: ハイパーパラメータを調整する（約800字）

- ハイパーパラメータとは何か（簡潔に）
- `GridSearchCV` で `n_estimators`, `max_depth` を探索
- `best_params_`, `best_score_` の確認
- 最終精度を出してStep1からの変化を振り返る
- **ゴール**: チューニングの基本フローを理解する

### 6. まとめ（約300字）

- 各ステップの精度変化を振り返り
- 次のステップへの案内（回帰タスク、他のモデル、パイプラインなど）
