## 概要

Pythonエンジニア向けのscikit-learn入門記事。Wineデータセットを使った分類タスクを題材に、段階的にステップアップしながら機械学習の基本フローを体験するチュートリアル。

## 前提条件

- Python 3.9以上
- scikit-learn 1.x（`pip install scikit-learn`）
- 全コード例で `random_state=42` を使用し、再現性を保証する

## アプローチ

**段階的ステップアップ型**: 最初は素朴な「fit → predict」から始めて、段階的に前処理・評価・チューニングを追加。各ステップで精度の変化を見せ、「なぜその工程が必要か」を体感させる。

## 記事構成

### 1. はじめに（約300字）

- scikit-learnとは何か（1〜2文で簡潔に）
- この記事で何ができるようになるか
- 使用するデータセット（Wine）の紹介
- 前提条件（Python 3.9+, scikit-learn 1.x）

### 2. Step1: まず動かす（約800字）

- `load_wine()` でデータ読み込み
- `train_test_split` で訓練データ・テストデータに分割（`random_state=42`）
- `SVC()` をデフォルト設定で `fit` → `predict`
- `accuracy_score` で精度確認（スケーリングなしなので0.6〜0.7程度を想定）
- **ゴール**: 最小限のコードで「動いた！」という成功体験。ただし精度はイマイチ

### 3. Step2: 前処理を入れる（約800字）

- 問題提起: Wineデータの特徴量はスケールがバラバラ（アルコール度数 vs プロリン含有量など）
- `Pipeline` で `StandardScaler` → `SVC` を構成して再学習
- 精度が大幅に改善（0.95+を想定）
- SVCはスケーリングの影響を強く受けるため、効果がはっきり見える
- Pipeline を使うことでデータリークも防止（ベストプラクティスとして紹介）
- **ゴール**: 前処理の重要性をスコアの変化で体感する

### 4. Step3: モデルを評価する（約800字）

- accuracyだけでは不十分という話（クラス不均衡の例を軽く触れる）
- `classification_report` でprecision, recall, f1-scoreを確認
- 混同行列は `confusion_matrix` の出力を `print` で表示（matplotlib不使用のため整形出力）
- 「どのクラスを間違えやすいか」を読み解く
- **ゴール**: 適切なモデル評価ができるようになる

### 5. Step4: ハイパーパラメータを調整する（約800字）

- ハイパーパラメータとは何か（簡潔に）
- `GridSearchCV` で `C`, `gamma`, `kernel` を探索（Pipeline内で実行 → データリーク防止）
- `best_params_`, `best_score_` の確認
- 最終精度を出してStep1からの変化を振り返る
- **ゴール**: チューニングの基本フローを理解する

### 6. まとめ（約300字）

- 各ステップの精度変化を振り返り（Step1: ~0.65 → Step2: ~0.97 → Step4: ~0.98）
- 次のステップへの案内（回帰タスク、他のモデル、クロスバリデーションの深掘りなど）

## 技術的判断

| 項目 | 決定 | 理由 |
|------|------|------|
| ライブラリ | sklearnのみ | コンパクトに収めるため。pandas, matplotlibは不使用 |
| データセット | Wine（3クラス分類、13特徴量） | 前処理の説明がしやすい。スケールのばらつきがある |
| モデル | SVC | スケーリングの効果が明確に出る。段階的改善の物語に適合 |
| Pipeline | Step2から導入 | データリーク防止。GridSearchCVとの組み合わせで正しいCV実行 |
| 再現性 | 全コードで `random_state=42` | 読者が同じ結果を再現できるようにする |
| 混同行列表示 | print出力 | matplotlib不使用の制約。classification_reportで十分に評価可能 |
| コードスタイル | 上から順に実行する想定 | 記事上はコードブロックを分けてステップごとに提示 |
| データ確認 | print + sklearn組み込み | pandas不使用のため |
